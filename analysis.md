### 6-4. 클라이언트 질문 답변

**Q1. React Native 기반 다국어 앱 개발 경험이 있으신가요? 지원 언어 수와 번역 처리 방식을 알려주세요.**

네, 보유하고 있습니다. React Native 단일 코드베이스로 iOS·Android·반응형 웹 3개 채널을 동시 출시한 프랜차이즈 관리 앱을 수행하였고, 별도 B2B SaaS 프로젝트에서는 한국어·영어 다국어 시스템을 구축한 경험이 있습니다.

번역 처리 방식은 두 가지로 이원화하여 설계합니다.
- 정적 UI 텍스트: i18n 리소스 번들(react-i18next)로 언어팩 관리 → 빠른 전환·오프라인 대응
- 동적 콘텐츠(제품 정보·실시간 상담): 번역 API 연동으로 자동 번역 후 결과 캐싱(Redis) → 다국어 환경 로딩 속도 확보
지원 언어는 영어·한국어를 기본으로 하되, 언어 추가 시 리소스 번들과 번역 파이프라인 확장만으로 대응 가능한 구조로 설계하여 글로벌 확장에 대비합니다. (구체적 지원 언어 수는 미팅 시 타겟 시장 기준으로 확정 제안)

**Q2. SNS 데이터 수집 API 연동 경험이 있으신가요? 있다면 구체적인 프로젝트 사례를 알려주세요.**

네, 외부 소셜/플랫폼 API 연동 경험을 다수 보유하고 있습니다.
- Connectin(글로벌 네트워킹 플랫폼): 외부 슈퍼앱(Zalo)의 OAuth·웹훅·메시징(ZNS)을 심층 연동하고, 소셜 피드·프로필 데이터를 수집·동기화하는 기능을 구축했습니다.
- P2P 크라우드펀딩 플랫폼: 7개 프로바이더·15개 이상 외부 API 접점(본인인증·가상계좌·정산·SMS)을 단일 시스템에 통합하였습니다.
SNS 팔로워·활동 데이터 수집 API(Phyllo 등)도 동일한 OAuth 인증 → 웹훅/폴링 수집 → 정규화·캐싱 패턴으로 안정적으로 연동 가능하며, 본 프로젝트의 5단계 등급 자동 산정 로직과 연계하여 구현하겠습니다.
